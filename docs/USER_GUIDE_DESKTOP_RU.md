# Руководство пользователя (Desktop)

## 1. Установка
1. Запустите один из установщиков:
   - `ParagraphTestSystem-Setup.exe` (Inno Setup, классический мастер),
   - `ParagraphTestSystem-Setup-IExpress.exe` (без Inno Setup, устанавливает в профиль пользователя).
2. Завершите мастер установки.
3. Запустите приложение из меню Пуск: `Paragraph Test System`.

## 2. Первый запуск
При старте приложение автоматически:
1. Поднимает внутренний backend.
2. Открывает интерфейс `/ui` во встроенном окне.

Если backend не стартует:
- Проверьте файл `backend\.env` в каталоге установки.
- Проверьте доступность `BACKEND_DB_DSN`.

## 3. Блок "Подключение" (верхняя панель)
Верхняя панель разделена на две зоны:
- **Подключение**
- **Авторизация**

### Подключение
- `Адрес системы тестирования (backend)` — обычно оставлять `http://127.0.0.1:8000`.
- `Профиль стенда` — выбирается целевой профиль (VM/хост/сборка).
- `Профили` — обновляет список профилей.
- `Автонастроить стенд` — пытается создать/обновить профиль автоматически.
- `Сохранить` — сохраняет адрес backend в локальные настройки UI.

### Авторизация
- `Логин` / `Пароль` — учетные данные пользователя Параграф.
- `Войти / Проверить` — выполняет проверку доступности (health/probe в зависимости от профиля).

## 4. Рекомендуемый рабочий цикл
1. Нажмите `Автонастроить стенд`.
2. Выберите профиль в `Профиль стенда`.
3. Нажмите `Войти / Проверить`.
4. Убедитесь, что `probe` успешен (ИШД/REST/DB).
5. Запускайте ручные операции или автотест.

## 5. Автотест "в одну кнопку"
1. Откройте секцию `Справочники (ИШД)`.
2. В блоке `Автотест справочников (одна кнопка)` выберите справочник.
3. При необходимости включите/выключите опции:
   - `create/insert/search/update/remove на временном справочнике`
   - `smoke create/delete по всем типам`
   - `подробные шаги`
4. Нажмите `Запустить автотест`.
5. Отслеживайте:
   - статус,
   - шаги,
   - историю прогонов,
   - JSON-отчет.

## 6. Работа с файлами в справочниках
1. Выполните `Search`/`Select Fields` и выберите строку.
2. Укажите `row_id`, колонку файла и `file_index`.
3. Для загрузки: выберите файл и нажмите `Загрузить файл`.
4. Для выгрузки: нажмите `Скачать файл`.

## 7. Диагностика типовых проблем

### 7.1 `Connection lost` по ИШД
- Проверить доступность внутренней сети/VPN.
- Проверить логин/пароль и профиль target.
- Проверить `POST /targets/{id}/probe` (в UI кнопкой проверки).

### 7.2 Ошибки автотеста
- Сначала запустить без CRUD-части.
- Проверить, что справочник доступен вручную (`search/query`).
- Сохранить `run_id` и JSON отчета для анализа.

### 7.3 Ошибки backend startup
- Проверить `backend\.env`.
- Проверить доступность БД backend по `BACKEND_DB_DSN`.

## 8. Адаптация под экраны и DPI
В desktop-версии включены:
- фиксированная минимальная геометрия окна,
- адаптивная сетка,
- устойчивые размеры шрифтов через `clamp`,
- мобильные/узкие breakpoints.

Рекомендации:
- Windows Scale: 100–150%.
- Минимальное окно: не ниже 1180x760 для полной рабочей области.

## 9. Логи и отчеты
- История автотестов: `backend\reports\autotest`.
- Отчеты доступны из UI (`Скачать полный JSON`).

## 10. Что передавать в баг-репорт
1. Дата/время прогона.
2. `run_id`.
3. JSON-отчет.
4. Скриншот UI с ошибкой.
5. Кратко: какой профиль target и какой шаг упал.
